# issue #54: プロファイル名空欄時にエラー無しで保存失敗する

## 原因(#53のバグ)

- `RepoSettingsView.vue` の `currentSettings()` が名前空欄のプロファイルを無言で
  送信対象から除外していた。
- サーバーの設定保存は「payloadに無い既存プロファイルは削除」の全量同期のため、
  既存プロファイルの名前を空欄にして保存すると削除される。
- 保存成功後にフォーム状態をサーバー応答で置き換えるため、入力中のカードも消える。

## 修正

- Web: 空欄フィルタを廃止し、名前空欄のプロファイルがあれば保存をブロックして
  エラーメッセージを表示する(何も無言で消えない)。
- サーバー: zodの`min(1)`は空白のみの名前を通すため、trim後に空になる名前を
  400で拒否する防御を追加。
- テスト: settings PUTルートのテストを追加(空欄名400 / 正常保存 / 同期削除)。
